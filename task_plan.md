## 桌面启动终端窗口修复计划

1. 修改 PyInstaller 打包参数
   - 在 `apps/api/build.ps1` 中加入 `--windowed` 或 `--noconsole`。
   - 同步修改 `apps/api/api-server.spec`，将 `console=True` 改为 `console=False`。

2. 增加无控制台后的日志策略
   - 因为用户看不到终端，后端启动失败时不能静默。
   - 在 `run_server.py` 或后端启动配置中把日志写入 `DC_DATA_DIR/logs/api-server.log`。
   - 至少记录启动端口、数据库路径、异常栈。

3. 重建 sidecar
   - 运行 `apps/api/build.ps1` 生成新的 `dist/api-server.exe`。
   - 将新的 sidecar 复制到 Tauri `externalBin` 对应位置。

4. 重建安装包
   - 运行 `apps/desktop` 下的 `npm run tauri:build`。
   - 生成新的 NSIS 安装包。

5. 验证
   - 双击安装后的桌面应用，确认不再出现终端窗口。
   - 确认前端能连接后端，`/health` 正常。
   - 确认关闭窗口后 `api-server.exe` 被 kill，不残留进程。
   - 模拟端口占用或后端启动失败，确认日志文件可用于排查。
